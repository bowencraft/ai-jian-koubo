# 审核页双模式融合时间线设计

**日期：** 2026-07-07  
**状态：** 设计已确认，等待书面 spec 复核  
**范围：** `scripts/templates/review.html`、`scripts/templates/review.css`、`scripts/templates/review.js` 及必要的共享波形渲染辅助逻辑

## 背景

AI剪口播已经从单素材审核流程升级为多素材时间线流程。当前审核页虽然能显示只读多轨项目，但 `project-tracks` 和 `waveCanvas` 仍然像两个独立视图：上方是多轨 clip 条，下方是转录混音波形和删减覆盖层。这个结构会让用户在审核时不断把“AI 删哪里”和“原始素材片段在哪条轨道上”做脑内对齐。

新的审核页需要把两者融合成一个符合常规剪辑软件认知的时间线：多轨 clip 是主体，波形和审核状态直接落在同一时间坐标里。用户要能快速看懂正常内容、最终删减内容、气口/停顿候选区域，并且可以在信息密度高低之间切换。

## 目标

- 把审核页的 `waveCanvas` 和 `project-tracks` 融合进同一个时间线视口，共用同一套缩放、平移、点击定位和播放头坐标。
- 提供两种可切换形式：
  - **叠加波形：** `waveCanvas` 透明叠加在 `project-tracks` 上方，显示混音波形和审核状态。
  - **裁切条：** `waveCanvas` 缩成很窄的一条状态条，不渲染混音波形，只显示删减/气口状态。
- 两种形式下，`project-tracks` 的每个 clip 内部都显示对应源素材的波形。
- 修复审核页波形主题选择功能，让主题切换能立即影响普通波形视觉，同时不改变删减、气口、播放头这些语义颜色。
- 保持现有审核、回到 editor、重新转文字/智能裁切标记、FCPXML 导出和旧项目迁移行为不变。

## 非目标

- 不把审核页改成完整编辑器。移动 clip、修剪 clip、增删素材和轨道仍然回到 `editor.html` 完成。
- 不在默认 UI 中显示“额外切掉”状态。算法额外裁切区域如果以后需要排查，可以作为高级调试开关另做，默认隐藏。
- 不把真实 waveform 数组写入 `project.json`、`data.json`、审核包项目 JSON 或导出的项目 JSON。
- 不重写审核算法、`compute_keeps.js` 或 FCPXML 导出语义。

## 视觉结构

审核页时间线由一个统一的 `review-timeline` 视口承载：

1. 顶部保留时间标尺。
2. 标尺下方是融合后的主时间线区域。
3. 主区域内始终有 `project-tracks`，按轨道显示多条 clip。
4. `waveCanvas` 根据模式改变形态：
   - 叠加波形模式：覆盖整个轨道区域。
   - 裁切条模式：变为轨道上方或标尺下方的一条窄状态条。
5. 播放头是一根跨越可视时间线区域的细橙线。

所有横向位置都使用同一公式：

```text
x = (timelineTime - viewStart) * pxPerSec
```

这样标尺、clip、clip 内波形、删减区域、气口区域和播放头不会出现错位。

## 模式一：叠加波形

叠加波形模式适合用户需要同时看“混音整体声音形态”和“多轨素材结构”的审核场景。

- `project-tracks` 作为底层 DOM 结构，显示轨道、clip 和 clip 内源素材波形。
- `waveCanvas` 使用透明背景，绝对定位覆盖在 `project-tracks` 上。
- `waveCanvas` 绘制内容：
  - 混音波形：使用当前主题的普通波形颜色，低透明度，避免压过 clip 信息。
  - 气口区域：青蓝色半透明带。
  - 删减区域：红色半透明带，是最高优先级的审核状态。
  - 播放头：细橙线，始终最后绘制。
- 如果删减区域和气口区域重叠，红色删减优先显示。
- 正常保留区域不铺绿色，也不额外盖色；原始 clip 颜色和波形就是默认保留状态。

交互事件绑定在统一时间线容器上，而不是依赖 canvas 自身独占事件。这样 clip 的 hover、点击定位、状态区域点击和播放头定位都可以基于同一个坐标换算处理。

## 模式二：裁切条

裁切条模式适合用户想专注多轨素材结构，同时快速扫一眼哪些范围会被删的场景。

- `project-tracks` 是主要视觉层，clip 内仍然显示源素材波形。
- `waveCanvas` 高度约为 20-24px，作为窄状态条嵌在同一时间线视口内。
- 裁切条不渲染混音波形。
- 裁切条只绘制：
  - 红色删减范围。
  - 青蓝色气口/停顿候选范围。
  - 细橙色播放头。
- 裁切条与下方 tracks 共用同一横向滚动/缩放坐标，用户可以从条上点击定位或选择状态范围。

## 状态颜色语义

审核页只保留三个内容状态：

| 状态 | 颜色 | 含义 |
| --- | --- | --- |
| 正常 | 不额外铺色 | 默认保留内容，使用 clip 原色和普通波形颜色表示 |
| 删减 | 红色 | 最终会被删除的内容，是最重要的状态 |
| 气口 | 青蓝色 | 静音、停顿或可一键选择的气口候选区域 |

播放头使用细橙线。它是时间位置指示器，不进入状态图例，也不参与状态计算。

建议使用固定语义色：

```text
delete:  rgba(239, 68, 68, 0.42)
breath:  rgba(6, 182, 212, 0.34)
playhead: #f97316
```

主题选择只影响普通视觉层，例如背景、标尺、混音波形、clip 内波形和中性边框。主题不得改变删减、气口、播放头的语义颜色。

## Clip 内波形

每个 `.project-clip` 内添加一个只读 waveform canvas，用于显示该 clip 对应源素材的真实波形片段。

渲染规则：

- 使用 `clip.assetId` 找到素材 waveform peaks。
- 使用 `clip.sourceStart` 和 `clip.duration` 截取源素材波形范围。
- 使用 `clip.timelineStart` 和 `clip.duration` 映射到时间线位置。
- 波形绘制在 clip 内部，随 clip 宽度响应缩放变化。
- 对长项目只绘制可视范围内的 clip 波形，避免一次性绘制所有 canvas。
- 如果某个素材没有 waveform cache，clip 仍正常显示，只隐藏内部波形或显示轻量占位底纹。

clip 内波形是源素材波形，`waveCanvas` 的叠加波形是审核混音波形。两者表达不同信息：前者帮助识别素材片段，后者帮助对齐最终审核音频。

## 交互逻辑

### 模式切换

审核页底部时间线工具栏增加一个两段式切换控件：

```text
叠加波形 | 裁切条
```

- 默认使用叠加波形模式。
- 用户选择写入 `localStorage.reviewTimelineMode`。
- 页面刷新后恢复上次模式。
- 切换模式不改变当前播放时间、缩放级别、滚动位置、选中删除项或 transcript 状态。

### 缩放和平移

- 两种模式共用当前审核页已有的 `pxPerSec`、`viewStart`、`duration` 概念。
- 缩放后同步重绘标尺、tracks、clip 内波形和 `waveCanvas`。
- 横向平移后同步更新所有层的位置。
- 高 DPI 屏幕按 `devicePixelRatio` 调整 canvas 实际像素，CSS 尺寸保持稳定。

### 点击与选择

- 点击任意时间线空白区域：定位播放头。
- 点击 clip：定位到对应时间，并可以显示素材名、轨道、源时间范围等只读信息。
- 点击红色删减区域：联动到对应删减项或 transcript 片段。
- 点击青蓝色气口区域：选中该气口候选，方便用户确认或一键加入删减。
- 拖拽移动/修剪 clip 不在审核页处理，用户通过“回到编辑器”继续调整素材和轨道。

### 图例

图例保持轻量，只解释审核状态：

- 正常：中性样式，不铺色。
- 删减：红色。
- 气口：青蓝色。

播放头不进入图例。

## 数据流

审核页加载数据来源保持现状：

- `data.json`：转录结果、AI 识别出的删减/气口候选、审核状态。
- `project.json`：多素材时间线模型，包括 `assets[]` 和 `clips[]`。
- `waveform_cache.json` 或现有本地 waveform API：真实波形缓存。

页面初始化时：

1. 规范化项目模型，兼容旧项目迁移出的 `assets[0] + clips[0]`。
2. 加载混音 waveform，用于 `waveCanvas` 的叠加波形模式。
3. 按需加载各素材 waveform，用于 clip 内波形。
4. 从审核数据生成状态区间：
   - 已确认或当前有效的删除区间生成红色删减区间。
   - 静音/停顿候选生成青蓝色气口区间。
5. 统一交给时间线坐标系统渲染。

任何 waveform 大数组只存在本地缓存或运行时内存中，不写回项目交换文件。

## 主题选择修复

审核页现有主题选择要改成可靠的单一入口：

- 初始化时从 `localStorage.reviewWaveTheme` 读取主题 key。
- 如果 key 不存在或已经失效，回落到默认主题。
- `themeSelect` 改变时：
  - 更新 `localStorage.reviewWaveTheme`。
  - 更新 CSS 变量或当前主题对象。
  - 更新图例的中性/普通波形样式。
  - 触发标尺、混音波形、clip 内波形和状态层重绘。
- 语义色 `delete`、`breath`、`playhead` 不从主题中读取，避免用户切换主题后删减状态失去稳定含义。

## 兼容性

- 没有 `project.json` 的旧审核包继续显示单轨迁移结果。
- 没有素材 waveform cache 时，clip 条仍然可见，审核流程不阻塞。
- 没有气口候选数据时，只显示删减状态和正常内容。
- 没有确认删减时，不显示红色区间，正常内容保持原色。
- FCPXML 导出继续以原始多素材、多轨 clip 和审核 delete ranges 为准，不依赖屏幕渲染状态。

## 性能策略

- 使用 `requestAnimationFrame` 合并连续缩放、滚动、主题切换造成的重绘。
- `project-tracks` 只更新必要的 transform/尺寸，不在每一帧重建全部 DOM。
- clip 内 waveform 采用可视区域绘制；离屏 clip 可以延迟或清空 canvas。
- 长时间线下 `waveCanvas` 只绘制当前可视时间窗。
- canvas 尺寸变化集中在 resize、模式切换和缩放变化时处理。

## 错误处理与降级

- waveform cache 读取失败：记录控制台警告，显示 clip 和状态条，不影响审核操作。
- 单个素材 waveform 缺失：该 clip 不画内部波形，但保留 clip 名称、颜色和边界。
- 主题 key 无效：回落默认主题并覆盖本地存储。
- 浏览器 canvas 绘制异常：不阻塞 transcript、删除项确认和导出按钮。

## 测试策略

实现时需要覆盖以下验证：

- 静态检查：
  - `node --check scripts/templates/review.js`
  - 若抽出共享 helper，同步检查对应 helper 文件。
- 单元或轻量 DOM 测试：
  - 模式 key 读写和无效 key 回退。
  - 状态区间分类：正常不铺色，删减红色，气口青蓝色，删减覆盖气口。
  - 时间坐标映射：ruler、tracks、overlay 使用同一 `viewStart` 和 `pxPerSec`。
- 浏览器验证：
  - 叠加波形模式下，`waveCanvas` 覆盖 tracks，混音波形、红色删减、青蓝气口和 clip 内波形同时可见。
  - 裁切条模式下，窄条不渲染混音波形，只显示红色/青蓝状态和橙色播放头。
  - 切换主题后，普通波形颜色变化，红色删减、青蓝气口、橙色播放头保持不变。
  - 切换模式后，当前播放时间、缩放和选中状态不丢失。
  - 旧单素材项目仍能迁移显示。
  - 多素材多轨项目导出 FCPXML 行为不变。

## 实现边界

这份设计建议采用“DOM tracks + canvas overlays”的混合架构：

- 保留 `project-tracks` 作为多轨 clip 的 DOM 结构。
- 为每个可见 clip 增加内部 waveform canvas。
- 保留 `waveCanvas`，但改成模式化 overlay/status strip。
- 抽出或复用 editor 中成熟的 waveform 绘制逻辑，避免在审核页重新发明一套 peak 绘制规则。
- 把时间坐标、主题应用、状态区间绘制和 clip waveform 绘制拆成小函数，便于测试和后续维护。
